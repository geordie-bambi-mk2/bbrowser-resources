Architecture

  • Extension popup: compact runtime status dashboard.
  • Options page: full-size settings editor with tabbed controls.
  • Helper app: local HTTP service on 127.0.0.1:5655.
  • VLC playback control and live timing sourced from helper /status.

Playback Lifecycle

  1. User opens a video on an allowed domain (or manual trigger target).
  2. Extension validates blacklist and domain mode constraints.
  3. URL is sent to helper with multi_monitor and input_lock flags.
  4. Helper launches VLC instance(s), sets up RC status channel, and tracks session.
  5. Extension status UI updates from helper playback data.

Domain Modes

  • AUTO: proactive detection on page load/navigation.
  • MANUAL: intent-based click/play pairing before hijack.
  • Per-domain learning and hinting improve manual mode targeting.
  • Blacklist takes priority and suppresses hijack for matching URLs.

Input Lock

  • Independent from multi-monitor mode.
  • Can be enabled manually or forced during lockdown timer.
  • Lockdown blocks disabling until timer expiry.
  • After expiry, setting remains enabled but becomes user-toggleable again.

Auto-Play Fallback

  • Enabled only when Bambi Mode is active.
  • If idle timeout is reached without VLC send, fallback URL opens.
  • Successful VLC send cancels fallback for active session context.
  • Best used with fallback pages on allowed domains.

Status Surface

  • Helper connectivity status (online/offline).
  • Multi-monitor and input-lock state reflection.
  • Playback state + elapsed / total / remaining timeline.
  • Progress bar derived from position fraction.
